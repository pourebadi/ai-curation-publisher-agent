MVP TELEGRAM TOPIC WORKFLOW TEST GUIDE

Goal
- Test one central bot.
- Test one source topic per category.
- Test multiple language outputs routed to separate review topics and final channels.
- Start with manual source messages first. Enable providers/media processor only after manual path passes.

Required before testing
1. Bot is added as admin to the internal forum group.
2. Bot can read messages in the source topic.
3. Bot can post in each review topic.
4. Bot is admin in each final public channel.
5. You know these numeric values:
   - source chat ID
   - source topic/thread ID
   - review chat ID
   - review topic/thread ID per language
   - your numeric Telegram user ID for TELEGRAM_ALLOWED_REVIEWER_IDS
   - final channel ID or @username per language

Minimum Worker secrets
- INTERNAL_API_SECRET
- TELEGRAM_BOT_TOKEN
- TELEGRAM_ALLOWED_REVIEWER_IDS

Useful safe Worker vars for first manual test
- TELEGRAM_REAL_REVIEW_ENABLED=true
- TELEGRAM_FINAL_PUBLISH_ENABLED=false
- TELEGRAM_PUBLISH_SCHEDULER_ENABLED=false
- AI_PROVIDER=mock
- MEDIA_PROCESSING_MODE=telegram_file_id_reuse

Set webhook
Run locally after deploy:

WORKER_BASE_URL=https://your-worker.workers.dev \
TELEGRAM_BOT_TOKEN=123:abc \
pnpm telegram:set-webhook

Create route config fast
Option A: Dashboard
- Open dashboard.
- Save Worker URL and internal secret.
- Go to Settings -> Telegram.
- Use MVP route builder.
- Save route.
- Run Check route config.

Option B: CLI smoke setup

WORKER_BASE_URL=https://your-worker.workers.dev \
INTERNAL_API_SECRET=your-secret \
TEST_REVIEWER_ID=123456789 \
SOURCE_CHAT_ID=-1001111111111 \
SOURCE_THREAD_ID=101 \
REVIEW_CHAT_ID=-1001111111111 \
REVIEW_THREAD_ID=201 \
FINAL_CHAT_ID=@crypto_fa \
TEST_LANGUAGE=fa \
pnpm telegram:mvp-smoke

Manual test flow
1. Send a text/link/photo/video message into the configured source topic.
2. Confirm a review draft appears in the configured review topic.
3. Press Status. It should answer only for that generated output.
4. Press Send. With final publishing disabled, it should queue/schedule but not publish.
5. In dashboard Activity, load outputs and queue.
6. Enable final publishing only after queue behavior is correct:
   TELEGRAM_FINAL_PUBLISH_ENABLED=true
7. If using scheduled mode, enable:
   TELEGRAM_PUBLISH_SCHEDULER_ENABLED=true
8. To force due processing manually, use Activity -> Run due publish or POST /internal/telegram/publish/due.

Queue behavior to verify
- Approve several FA review drafts quickly. They should enter only the FA final channel queue.
- Approve AR or EN separately. Their queues should not mix with FA.
- Scheduler publishes at most one due item per final destination per run.
- minimumGapMinutes and allowedPublishWindows are applied per route output.

Optional GitHub Actions media processor
Use this only after the manual Telegram-native flow works.

Worker settings/secrets:
- MEDIA_PROCESSING_MODE=github_actions
- GITHUB_MEDIA_PROCESSOR_ENABLED=true
- GITHUB_MEDIA_PROCESSOR_REPOSITORY=owner/repo
- GITHUB_MEDIA_PROCESSOR_WORKFLOW_ID=media-processor.yml
- GITHUB_MEDIA_PROCESSOR_REF=main
- GITHUB_MEDIA_PROCESSOR_CALLBACK_URL=https://your-worker.workers.dev/internal/media/processing/callback
- GITHUB_MEDIA_PROCESSOR_TOKEN as Worker secret, with workflow dispatch permission

GitHub Actions secrets in the media processor repository:
- TELEGRAM_BOT_TOKEN
- TELEGRAM_MEDIA_CACHE_CHAT_ID or TELEGRAM_MEDIA_STAGING_CHAT_ID
- TELEGRAM_MEDIA_CACHE_THREAD_ID optional
- WORKER_INTERNAL_API_SECRET or INTERNAL_API_SECRET
- INSTAGRAM_COOKIES_B64 optional
- X_COOKIES_B64 optional

Current MVP media behavior
- Telegram-native media from source messages uses Telegram file_id reuse.
- External direct media and many public social URLs are attempted through GitHub Actions using yt-dlp/requests.
- Video thumbnail generation is attempted with ffmpeg.
- Oversized media is recompressed where possible, then rejected with a clear failed status if still too large.
- Deep login-gated X/Instagram extraction may need cookies.
- Complex carousels may still need provider-specific work after MVP validation.
